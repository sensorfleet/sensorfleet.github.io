Capture Engine Instrument

Overview

Capture Engine Instrument is used to mirror traffic from one physical interface to one or more mirror-bridge interfaces. Using Capture Engine allows multiple Instruments to have access to same traffic flow without the need for them to all have access to the physical interface.

Using Capture Engine separates the physical interface from any Instrument actually parsing the traffic, thus reducing possibility that Instrument could access the network through physical interface.

Instrument properties

Instrument developer SensorFleet Oy
Categories Internal
Network access type Passive
Required interfaces The physical interface whose data is to be copied (phy with receive capabilities), Interface(s) where to mirror the data from physical interface (mirror-bridge with send capabilities)
Instrument dependencies None
Data retention Capture Engine does not store information
Management UI No
Performance Capture Engine should be able to mirror gigabits/sec of traffic with F-Probe hardware
